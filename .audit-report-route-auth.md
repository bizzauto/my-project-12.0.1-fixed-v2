## CRITICAL: src/server/routes/sso.ts

**Auth middleware used?** NO
**Which auth?** None whatsoever
**Risk level:** CRITICAL
**Mount in index.ts:** /api/sso -- no auth middleware (line 390)

All 6 routes are completely unprotected:

| Method | Route | Description | Sensitive Data |
|--------|-------|-------------|----------------|
| GET | /api/sso | List SSO configs | Provider configs |
| POST | /api/sso | Create SSO config | clientId, clientSecret |
| PUT | /api/sso/:id | Update SSO config | clientSecret |
| DELETE | /api/sso/:id | Delete SSO config | -- |
| GET | /api/sso/auth/:provider | Initiate SSO login | OAuth state |
| GET | /api/sso/callback/:provider | OAuth callback handler | Auth codes |

**Vulnerability note:** The code reads (req as any).businessId which is undefined without authenticate middleware. GET queries return empty results because businessId is undefined. However, POST/PUT routes allow creating/updating SSO configs (including clientSecret) with an attacker-supplied businessId in the request body. An attacker could inject SSO configs into existing businesses.

**Recommendation:** Add authenticatedCsrf wrapper in index.ts:
  app.use('/api/sso', authenticatedCsrf, ssoRoutes);
Or add authenticate to each route handler inside sso.ts.

## Recommendations

### 1. CRITICAL: Fix sso.ts (src/server/routes/sso.ts)

Add authentication middleware to all SSO routes. This is the only route file handling sensitive credentials (client secrets) with zero authentication.

**Option A** -- Add authenticatedCsrf in index.ts (line 390):
  app.use('/api/sso', authenticatedCsrf, ssoRoutes);

**Option B** -- Add authenticate to each route handler in sso.ts.

### 2. CONSIDER: Strengthen dograh-webhook.ts Custom Auth

Make HMAC signature verification mandatory for businesses with Dograh integration enabled. Currently the check is optional (skipped if secret is null). Consider adding rate limiting.

### 3. INFORMATIONAL: Rate Limit store-public.ts

The order creation endpoint (POST /api/store/:businessId/orders) is fully public and can create contacts and process payments. Consider adding rate limiting and abuse prevention.

### 4. INFORMATIONAL: Refactor white-label.ts

The custom wlAuth middleware duplicates JWT verification logic from auth.ts. Consider reusing the main authenticate middleware with an additional role/permission check for resellers.
