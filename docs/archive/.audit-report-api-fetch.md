## Summary

The project provides a centralized apiClient (axios-based) in src/lib/api.ts that auto-injects:
- JWT Bearer token (from localStorage) on every request
- CSRF tokens on state-changing methods (POST, PUT, PATCH, DELETE)
- Refresh token rotation on 401 responses
- Consistent error handling via interceptors

The files below use raw fetch() instead, bypassing all of the above. Most are authenticated backend endpoints that will fail silently (or behave unpredictably) when the user's session token is required.

## Key Findings

1. **No overlap**: None of the 37 files using raw fetch() also import the official apiClient from ../lib/api (except NetworkStatus.tsx which lazily requires it but still uses fetch() directly).

2. **Manual token handling in some files**: LeadGenerationPage.tsx and ResellerDashboardPage.tsx manually read localStorage.getItem('token') and pass Authorization headers. This is fragile and misses apiClient benefits (refresh token rotation, CSRF tokens, unified error handling).

3. **Missing auth in most files**: The majority of files (30 out of 37) use raw fetch() with no Authorization header at all, relying on cookies or luck. These will fail for authenticated requests.

4. **BlogManager.tsx is the worst offender**: Defines an entire API object (blogAPI) with 18 raw fetch() calls but no auth headers and no apiClient usage.

5. **ava.service.ts uses manual auth**: Correctly attaches Bearer token via getHeaders(), but still bypasses apiClient's CSRF injection, refresh rotation, and global error handling.

---

## Recommendations

1. Migrate all raw fetch() calls to apiClient from src/lib/api.ts
2. For ava.service.ts: Replace fetch() with apiClient and remove manual getHeaders()
3. For BlogManager.tsx: Replace inline blogAPI with apiClient calls
4. For LeadGenerationPage.tsx: Replace manual localStorage token + fetch() with apiClient
5. Priority: Fix HIGH first, then MEDIUM, then LOW
